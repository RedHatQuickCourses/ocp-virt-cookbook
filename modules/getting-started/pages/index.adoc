== Overview

== Who Is This For?

Different roles can take different paths through this module.

VM administrator migrating from VMware or Hyper-V::
Start with xref:prerequisites.adoc[] to understand the environment. Then follow xref:install-openshift-virtualization.adoc[] and xref:create-vm-web-console.adoc[] to create your first VM from the web console. Continue with xref:vm-lifecycle-states.adoc[] to learn how VMs are managed in OpenShift.

Developer needing a test VM::
Start with xref:vm-quick-start.adoc[] to get a VM running in minutes. Then read xref:virtctl-basics.adoc[] to manage VMs from the command line.

Platform engineer::
Follow the full learning path in order: xref:prerequisites.adoc[], xref:install-openshift-virtualization.adoc[], xref:default-storage-class.adoc[], and both VM creation tutorials. After completing this module, continue with the xref:networking:index.adoc[Networking] and xref:storage:index.adoc[Storage] modules.

Evaluator or decision maker::
Read xref:install-openshift-virtualization.adoc[] to understand what the operator provides. Then follow xref:create-vm-web-console.adoc[] to see a VM running in the console. The xref:vm-lifecycle-states.adoc[] page shows day-to-day management operations.

== What You Will Learn

In this section, you will learn:

* Prerequisites and required tools for working with OpenShift Virtualization
* The architecture of OpenShift Virtualization and how VMs run on Kubernetes
* Basic usage of the virtctl command-line tool for managing virtual machines
* How to install the OpenShift Virtualization operator
* How to configure default storage for virtual machine deployments
* How to create virtual machines from the web console and from the CLI
* VM lifecycle states and management operations
* Basic usage of the virtctl command-line tool for managing virtual machines
* Cloud-init fundamentals and VM initialization patterns

== Prerequisites

Before proceeding with the tutorials in this section, ensure you have:

* An OpenShift 4.18+ cluster with OpenShift Virtualization operator installed
* Cluster admin access or appropriate RBAC permissions
* CLI tools installed: `oc`, `kubectl`, `virtctl`
* Basic understanding of Kubernetes/OpenShift concepts
