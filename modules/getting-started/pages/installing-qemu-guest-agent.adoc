= Installing QEMU Guest Agent
:navtitle: Installing QEMU Guest Agent
:page-toclevels: 3

== Overview

This tutorial demonstrates how to install the qemu-guest-agent in various Linux distributions. The QEMU guest agent is a daemon that runs inside the guest OS. It enables application-consistent snapshots, guest OS and filesystem reporting, dynamic SSH-key and password injection, IP address reporting, and exec/ping health probes.

== Prerequisites

* **OpenShift 4.18+** with OpenShift Virtualization operator installed (see xref:create-vm-web-console.adoc[Installing OpenShift Virtualization])
*  CLI tools: `oc`, `virtctl`


Versions tested:
[source,bash]
----
OpenShift 4.21, 4.22
----

Create a namespace where all the virtual machines for this tutorial will live.
[source,bash,role=execute]
----
oc create namespace qemu-ga-demo
----

[source,bash,role=execute]
----
oc project qemu-ga-demo
----

== Install via `virtctl console`

In this section we use `virtctl console <vm-name>` to connect to the console of a virtual machine, log in as the user created by cloud-init, install `qemu-guest-agent` using the distribution's package manager, and enable the service with `systemctl enable --now qemu-guest-agent` to start it and persist it across reboots.

WARNING: The cloud-init passwords in the following examples are for demo purposes only. Always use a strong, unique password or SSH key authentication in production environments.

=== Fedora/RHEL

==== Step 1: Create the Fedora virtual machine

xref:attachment$installing-qemu-guest-agent/fedora-no-qga.yaml[Download fedora-no-qga.yaml]
[source,yaml,role=execute]
----
apiVersion: kubevirt.io/v1
kind: VirtualMachine
metadata:
  name: fedora-virtctl
  namespace: qemu-ga-demo
spec:
  runStrategy: Always
  dataVolumeTemplates:
    - metadata:
        name: fedora-virtctl-rootdisk
      spec:
        storage:
          resources:
            requests:
              storage: 10Gi
          accessModes:
            - ReadWriteOnce
        source:
          registry:
            url: docker://quay.io/ocpv_cookbook/fedora-no-qga:latest
  template:
    metadata:
      labels:
        app: fedora-virtctl
    spec:
      domain:
        cpu:
          cores: 2
        memory:
          guest: 4Gi
        devices:
          disks:
            - name: rootdisk
              disk:
                bus: virtio
            - name: cloudinitdisk
              disk:
                bus: virtio
      volumes:
        - name: rootdisk
          dataVolume:
            name: fedora-virtctl-rootdisk
        - name: cloudinitdisk
          cloudInitNoCloud:
            userData: |-
              #cloud-config
              user: fedora
              password: fedora
              chpasswd: { expire: false }
----

NOTE: We are using a datavolume instead of containerdisk, which increases time-to-first-boot but has the benefit of saving the root volume onto a PVC making the changes in the following steps persistent even if virt-launcher crashes or if the virtual machine is stopped and then started again.

Create the virtual machine from the CLI: (click <<fedora-create-web-console,here>> to create via the web console)

[source,bash,role=execute]
----
oc create -f fedora-no-qga.yaml
----

Wait for the virtual machine's STATUS to be `Running`:

[source,bash,role=execute]
----
oc get vm fedora-virtctl -w 
----

[source,bash]
----
# example output

----

== Install via cloud-init `packages` and `runcmd` modules

The `packages` module from cloud-init allows you to install packages during the boot process. In this section we use it to install the `qemu-guest-agent` package. Once installed, the cloud-init `runcmd` module runs `systemctl enable --now qemu-guest-agent` to start and enable the service during boot.

WARNING: The cloud-init passwords in the following examples are for demo purposes only. Always use a strong, unique password or SSH key authentication in production environments.

=== Fedora/RHEL

xref:attachment$installing-qemu-guest-agent/fedora-qga-package.yaml[Download fedora-qga-package.yaml]

----

  name: fedora-cloudinit
  namespace: qemu-ga-demo
spec:
  runStrategy: Always
  dataVolumeTemplates:
    - metadata:
        name: fedora-cloudinit-rootdisk
      spec:
        storage:
          resources:
            requests:
              storage: 10Gi
          accessModes:
            - ReadWriteOnce
        source:
          registry:
            url: docker://quay.io/ocpv_cookbook/fedora-no-qga:latest
  template:
    metadata:
      labels:
        app: fedora-cloudinit
    spec:
      domain:
        cpu:
          cores: 2
        memory:
          guest: 4Gi
        devices:
          disks:
            - name: rootdisk
              disk:
                bus: virtio
            - name: cloudinitdisk
              disk:
                bus: virtio
      volumes:
        - name: rootdisk
          dataVolume:
            name: fedora-cloudinit-rootdisk
        - name: cloudinitdisk
          cloudInitNoCloud:
            userData: |-
              #cloud-config
              user: fedora
              password: fedora
              chpasswd: { expire: false }
              packages: <.>
                - qemu-guest-agent
              runcmd: <.>

<.> Use the **packages** cloud-init module to install `qemu-guest-agent`
<.> Use the **runcmd** cloud-init module to run any command during boot

[#verify-qga-reporting]
== Verifying `qemu-guest-agent` is Reporting to OCP-Virt

There are several ways to verify if the qemu-guest-agent is working and properly reporting information about the guest OS to OCP-Virt through the web console or through the terminal using the `oc` and `virtctl` CLI tools.

=== Verifying through the terminal

Check that the `AgentConnected` condition is `True`:

[source,bash,role=execute]
----
oc get vm $VM_NAME -oyaml | yq -e '.status.conditions[] | select(.type == "AgentConnected")'
----

A successful output would look something like this:

[source,yaml]
----
# example output
lastProbeTime: "2026-06-04T13:33:28Z"
lastTransitionTime: null
status: "True"
type: AgentConnected
----

Check that the guest-agent is reporting the VM's IP Address:

[source,bash,role=execute]
----
oc get vmi $VM_NAME -oyaml | yq -e '.status.interfaces[]'
----

[source,bash]
----
# example output
infoSource: domain, guest-agent <1>
interfaceName: enp1s0
ipAddress: 10.130.0.22
ipAddresses:
  - 10.130.0.22
linkState: up
mac: 0a:58:0a:82:00:16
name: default
podInterfaceName: eth0
queueCount: 1
----
<1> When `guest-agent` is present as an infoSource, it indicates that the IP Address is being reported by the qemu-guest-agent.

Retrieve detailed guest OS information:

[source,bash,role=execute]
----
virtctl guestosinfo $VM_NAME

In this tutorial you learned how to:

* Install and enable `qemu-guest-agent` manually via `virtctl console` on Fedora, Debian, and openSUSE Leap

* Verify the agent is connected and reporting guest OS information to OpenShift Virtualization
