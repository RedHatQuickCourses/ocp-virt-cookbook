= YAML Manifests Reference
:navtitle: Manifests

A collection of ready-to-use YAML manifests for OpenShift Virtualization.

== Overview

This section provides a library of YAML manifests organized by category:

* **Operators** - LVM Storage, NMState, OVS bridge configuration
* **Networking** - NADs, UDNs, Linux bridges, overlay networks
* **Storage** - LVMCluster, StorageClass, PVCs, DataVolumes
* **Virtual Machines** - Complete VM templates for Fedora, RHEL, Windows
* **Security** - Network policies, RBAC roles, service accounts

== Quick Start

. Browse the xref:manifest-library.adoc[Manifests Library] to find the manifest you need
. Copy the YAML and customize values for your environment
. Apply with `oc apply -f manifest.yaml`
. Verify with appropriate `oc get` commands

== Customization Tips

Before applying manifests, update these common fields:

* `metadata.namespace` - Target namespace
* `metadata.name` - Resource name
* Device paths - Block devices for storage
* VLAN IDs - Network VLAN tags
* IP ranges - Subnets and addresses
* Interface names - Physical network interfaces

== Validation

[source,bash,role=execute]
----
# Validate syntax before applying
oc apply --dry-run=client -f manifest.yaml

# Check API version exists
oc api-resources | grep <kind>

# Verify permissions
oc auth can-i create <resource>
----
