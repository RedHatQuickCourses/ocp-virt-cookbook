= Virtual Machine Configuration
:navtitle: VM Configuration

== Overview

This module covers comprehensive virtual machine configuration for OpenShift Virtualization, with a focus on cloud-init for network and system initialization. Proper VM configuration is essential for deploying production-ready virtual machines with predictable networking and system settings.

== What You'll Learn

In this section, you will learn:

* How to configure IP addresses with cloud-init
* How to troubleshoot cloud-init network configurations  
* Best practices for VM configuration
* Common VM manifest patterns and examples

== Why Cloud-init Matters

Cloud-init is the industry-standard tool for initializing cloud instances. In OpenShift Virtualization, cloud-init provides:

* **Network Configuration**: Configure static and dynamic IP addresses
* **User Management**: Create users, set passwords, configure SSH keys
* **System Initialization**: Install packages, run scripts, configure services
* **File Management**: Create and configure files during VM boot
* **Consistency**: Same configuration method across different OS images

Before proceeding with the tutorials in this section, ensure you have:

== Cloud-init Configuration Approaches

=== networkData vs userData

* **networkData**: Dedicated network configuration (preferred for network settings)
* **userData**: General cloud-init configuration (users, packages, scripts)

Both can be used together in the same VM definition.

=== When to Use Cloud-init

Use cloud-init when you need to:

* Configure network interfaces with specific IP addresses
* Set up VMs with predictable network configuration
* Initialize VMs with specific users and SSH keys
* Run initialization scripts at VM boot time
* Configure multiple network interfaces

== Common Use Cases

=== Development VMs

* Quick deployment with DHCP
* Temporary user accounts
* Minimal configuration

=== Production VMs

* Static IP configurations
* Multiple network interfaces for segregation
* SSH key-based authentication
* Service initialization scripts

=== Multi-Network VMs

* Primary interface on pod network (DHCP)
* Secondary interface on VLAN (static IP)
* Network segmentation for compliance
