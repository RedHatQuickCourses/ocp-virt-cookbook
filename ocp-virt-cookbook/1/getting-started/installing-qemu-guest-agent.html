Installing QEMU Guest Agent

Overview

This tutorial demonstrates how to install the qemu-guest-agent in various Linux distributions. The QEMU guest agent is a daemon that runs inside the guest OS. It enables application-consistent snapshots, guest OS and filesystem reporting, dynamic SSH-key and password injection, IP address reporting, and exec/ping health probes.

What You’ll Learn

  • How to install and enable qemu-guest-agent manually via virtctl console on Fedora, Debian, and openSUSE Leap

  • How to install qemu-guest-agent automatically using cloud-init packages and runcmd modules

  • How to verify the agent is connected and reporting guest OS information to OpenShift Virtualization

Prerequisites

  • OpenShift 4.18+ with OpenShift Virtualization operator installed (see Installing OpenShift Virtualization)

  • CLI tools: oc, virtctl, yq

Versions tested:

OpenShift 4.21, 4.22

Setting Up the Namespace

Create a namespace where all the virtual machines for this tutorial will live.

oc create namespace qemu-ga-demo
oc project qemu-ga-demo

Install via virtctl console

In this section we use virtctl console <vm-name> to connect to the console of a virtual machine, log in as the user created by cloud-init, install qemu-guest-agent using the distribution’s package manager, and enable the service with systemctl enable --now qemu-guest-agent to start it and persist it across reboots.

The cloud-init passwords in the following examples are for demo purposes only. Always use a strong, unique password or SSH key authentication in production environments.

Fedora/RHEL

Step 1: Create the Fedora virtual machine

Create the virtual machine from the CLI: (click here to create via the web console)

We are using a datavolume instead of containerdisk, which increases time-to-first-boot but has the benefit of saving the root volume onto a PVC making the changes in the following steps persistent even if virt-launcher crashes or if the virtual machine is stopped and then started again.

Download fedora-no-qga.yaml

oc apply -f - <<EOF
apiVersion: kubevirt.io/v1
kind: VirtualMachine
metadata:
  name: fedora-virtctl
  namespace: qemu-ga-demo
spec:
  runStrategy: Always
  dataVolumeTemplates:
    - metadata:
        name: fedora-virtctl-rootdisk
      spec:
        storage:
          resources:
            requests:
              storage: 10Gi
          accessModes:
            - ReadWriteOnce
        source:
          registry:
            url: docker://quay.io/ocpv_cookbook/fedora-no-qga:latest
  template:
    metadata:
      labels:
        app: fedora-virtctl
    spec:
      domain:
        cpu:
          cores: 2
        memory:
          guest: 4Gi
        devices:
          disks:
            - name: rootdisk
              disk:
                bus: virtio
            - name: cloudinitdisk
              disk:
                bus: virtio
      volumes:
        - name: rootdisk
          dataVolume:
            name: fedora-virtctl-rootdisk
        - name: cloudinitdisk
          cloudInitNoCloud:
            userData: |-
              #cloud-config
              user: fedora
              password: fedora
              chpasswd: { expire: false }
EOF

Wait for the virtual machine’s STATUS to be Running:

oc get vm fedora-virtctl -w
The -w flag watches continuously. Press Ctrl+C once the STATUS shows Running. 
# example output
NAME                AGE    STATUS    READY
fedora-virtctl           6h8m   Running   True

Continue to step 2

Create the virtual machine from the web console:

  1. Navigate to Virtualization → VirtualMachines

  2. Click on the dropdown next to the Create button on the top right

  3. Select With YAML

  4. Select all in the text editor window and delete

  5. Paste the YAML provided above

  6. Click Create on the bottom left

You will be redirected to the "Overview tab" of your newly created VM. Wait for the Status to be Running.

Step 2: Install and enable qemu-guest-agent

From the CLI: (click here for Web console steps)

virtctl console fedora-virtctl

You might have to press ENTER to get the log-in prompt to print out. Once it appears enter fedora for both the username and password.

continue to the next step

From the Web console:

  1. Navigate to Virtualization → VirtualMachines

  2. Click on the name of virtual machine

  3. In the Details panel of overview tab click "Open Web Console" (this will open a new browser window)

  4. Type in fedora for your username, press enter

  5. Type in fedora for your password, press enter

Verify that qemu-guest-agent is not installed.

systemctl status qemu-guest-agent
# example output

Install and enable qemu-guest-agent using dnf:

sudo dnf install -y qemu-guest-agent && sudo systemctl enable --now qemu-guest-agent

Install via cloud-init packages and runcmd modules

The packages module from cloud-init allows you to install packages during the boot process. In this section we use it to install the qemu-guest-agent package. Once installed, the cloud-init runcmd module runs systemctl enable --now qemu-guest-agent to start and enable the service during boot.

The cloud-init passwords in the following examples are for demo purposes only. Always use a strong, unique password or SSH key authentication in production environments.

Fedora/RHEL

Create the virtual machine:

Download fedora-qga-package.yaml

oc apply -f - <<EOF
apiVersion: kubevirt.io/v1
kind: VirtualMachine
metadata:
  name: fedora-cloudinit
  namespace: qemu-ga-demo
spec:
  runStrategy: Always
  dataVolumeTemplates:
    - metadata:
        name: fedora-cloudinit-rootdisk
      spec:
        storage:
          resources:
            requests:
              storage: 10Gi
          accessModes:
            - ReadWriteOnce
        source:
          registry:
            url: docker://quay.io/ocpv_cookbook/fedora-no-qga:latest
  template:
    metadata:
      labels:
        app: fedora-cloudinit
    spec:
      domain:
        cpu:
          cores: 2
        memory:
          guest: 4Gi
        devices:
          disks:
            - name: rootdisk
              disk:
                bus: virtio
            - name: cloudinitdisk
              disk:
                bus: virtio
      volumes:
        - name: rootdisk
          dataVolume:
            name: fedora-cloudinit-rootdisk
        - name: cloudinitdisk
          cloudInitNoCloud:
            userData: |-
              #cloud-config
              user: fedora
              password: fedora
              chpasswd: { expire: false }
              packages: (1)
                - qemu-guest-agent
              runcmd: (2)
                - systemctl enable --now qemu-guest-agent (3)
EOF
1 Use the packages cloud-init module to install qemu-guest-agent
2 Use the runcmd cloud-init module to run any command during boot
3 Run systemctl enable --now qemu-guest-agent after the package has been installed to start the service

Wait for the virtual machine’s STATUS to be Running:

oc get vm fedora-cloudinit -w
The -w flag watches continuously. Press Ctrl+C once the STATUS shows Running. 
# example output
NAME                     AGE     STATUS    READY
fedora-cloudinit   8m23s   Running   True

Set the VM_NAME variable so the verification commands work for this VM:

VM_NAME=fedora-cloudinit

Proceed to Verifying qemu-guest-agent is Reporting to OCP-Virt to confirm the agent is connected and reporting to OCP-Virt.

Verifying qemu-guest-agent is Reporting to OCP-Virt

There are several ways to verify if the qemu-guest-agent is working and properly reporting information about the guest OS to OCP-Virt through the web console or through the terminal using the oc and virtctl CLI tools.

Verifying through the terminal

Check that the AgentConnected condition is True:

oc get vm $VM_NAME -oyaml | yq -e '.status.conditions[] | select(.type == "AgentConnected")'

A successful output would look something like this:

# example output
lastProbeTime: "2026-06-04T13:33:28Z"
lastTransitionTime: null
status: "True"
type: AgentConnected

Check that the guest-agent is reporting the VM’s IP Address:

oc get vmi $VM_NAME -oyaml | yq -e '.status.interfaces[]'
# example output
infoSource: domain, guest-agent (1)
interfaceName: enp1s0
ipAddress: 10.130.0.22
ipAddresses:
  - 10.130.0.22
linkState: up
mac: 0a:58:0a:82:00:16
name: default
podInterfaceName: eth0
queueCount: 1
1 When guest-agent is present as an infoSource, it indicates that the IP Address is being reported by the qemu-guest-agent.

Retrieve detailed guest OS information:

virtctl guestosinfo $VM_NAME

Verifying through the OCP-Virt web console

  1. Navigate to Virtualization → VirtualMachines

  2. Click on your virtual machine

In the Overview tab you can see the qemu-guest-agent reporting the guest OS information:

In the Diagnostics tab you can see the virtual machine has AgentConnected with a status of True:
